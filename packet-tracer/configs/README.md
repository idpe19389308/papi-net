# Διαμορφώσεις Cisco Packet Tracer

Φάκελος με τις τελικές διαμορφώσεις (running-configs) των πέντε δρομολογητών και πέντε μεταγωγέων που συγκροτούν τη δικτυακή υποδομή της εργασίας.

## Περιεχόμενα φακέλου

### Δρομολογητές (Routers)

| Αρχείο | Συσκευή | Διεπαφές WAN |
|---|---|---|
| [`R-IOA.txt`](R-IOA.txt) | Δρομολογητής Ιωαννίνων | προς THE και PAT |
| [`R-THE.txt`](R-THE.txt) | Δρομολογητής Θεσσαλονίκης | προς IOA και ATH |
| [`R-ATH.txt`](R-ATH.txt) | Δρομολογητής Αθήνας | προς THE και CHA |
| [`R-PAT.txt`](R-PAT.txt) | Δρομολογητής Πάτρας | προς IOA και CHA |
| [`R-CHA.txt`](R-CHA.txt) | Δρομολογητής Χανίων | προς ATH και PAT |

### Μεταγωγείς (Switches)

| Αρχείο | Συσκευή |
|---|---|
| [`SW-IOA.txt`](SW-IOA.txt) | Μεταγωγέας Ιωαννίνων (τρία VLAN) |
| [`SW-THE.txt`](SW-THE.txt) | Μεταγωγέας Θεσσαλονίκης |
| [`SW-ATH.txt`](SW-ATH.txt) | Μεταγωγέας Αθήνας |
| [`SW-PAT.txt`](SW-PAT.txt) | Μεταγωγέας Πάτρας |
| [`SW-CHA.txt`](SW-CHA.txt) | Μεταγωγέας Χανίων |

## Εξοπλισμός στο Packet Tracer

| Κατηγορία | Πλήθος | Μοντέλο |
|---|---|---|
| Δρομολογητές | 5 | Cisco PT8200 (4 × GigabitEthernet) |
| Μεταγωγείς | 5 | Cisco 2960-24TT |
| Σταθμοί εργασίας | 15 | Generic PC-PT (τρεις ανά παράρτημα, ένας ανά τμήμα) |
| Εξυπηρετητές | 2 | Generic PC-PT στο παράρτημα Ιωαννίνων (Web server + File server) |

### Χρήση διεπαφών δρομολογητή

| Διεπαφή | Ρόλος |
|---|---|
| `GigabitEthernet0/0/0` | LAN trunk προς τον μεταγωγέα (router-on-a-stick με VLAN subinterfaces `.10`/`.20`/`.30`) |
| `GigabitEthernet0/0/1` | WAN προς γείτονα Α |
| `GigabitEthernet0/0/2` | WAN προς γείτονα Β |
| `GigabitEthernet0/0/3` | Εφεδρική (ασύνδετη) |

## Φόρτωση διαμόρφωσης σε συσκευή

1. Διπλό κλικ στη συσκευή (router ή switch) στο workspace του Packet Tracer.
2. Επιλογή της καρτέλας **CLI**.
3. Πατήστε `Enter` για εμφάνιση του prompt.
4. Στην ερώτηση *"Continue with configuration dialog?"* απαντήστε `no`.
5. Πληκτρολογήστε:
   ```
   enable
   configure terminal
   ```
6. Αντιγραφή του πλήρους περιεχομένου του αντίστοιχου αρχείου `.txt` (από `hostname` έως `end`) και επικόλληση στο παράθυρο CLI.
7. Μετά την ολοκλήρωση, αποθήκευση της διαμόρφωσης ώστε να επιβιώσει σε επανεκκίνηση:
   ```
   copy running-config startup-config
   ```

## Προτεινόμενη σειρά ρύθμισης

1. Πρώτα ρυθμίζονται όλοι οι μεταγωγείς (SW-IOA, SW-THE, SW-ATH, SW-PAT, SW-CHA).
2. Στη συνέχεια ρυθμίζονται όλοι οι δρομολογητές (R-IOA, R-THE, R-ATH, R-PAT, R-CHA).
3. Στους σταθμούς εργασίας ορίζονται χειρωνακτικά οι διευθύνσεις IP, η μάσκα δικτύου και η προεπιλεγμένη πύλη μέσω της επιλογής *Desktop → IP Configuration*.

## Σχήμα VLAN (ίδιο σε όλα τα παραρτήματα)

| VLAN | Τμήμα | Υποδίκτυο | Διεύθυνση πύλης |
|---|---|---|---|
| 10 | Λογιστήριο | `.0/26` | `.1` |
| 20 | Διεύθυνση | `.64/26` | `.65` |
| 30 | Software Developers | `.128/26` | `.129` |

## Παράδειγμα διευθυνσιοδότησης σταθμών εργασίας (παράρτημα Αθήνας, LAN 200.1.100.0/24)

| Σταθμός | Διεύθυνση IP | Μάσκα | Πύλη |
|---|---|---|---|
| Σταθμός Λογιστηρίου | `200.1.100.10` | `255.255.255.192` | `200.1.100.1` |
| Σταθμός Διεύθυνσης | `200.1.100.74` | `255.255.255.192` | `200.1.100.65` |
| Σταθμός Software Developers | `200.1.100.138` | `255.255.255.192` | `200.1.100.129` |
