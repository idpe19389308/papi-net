============================================================
ER Diagram — PAPI-net (Διαχείριση Παραγγελιών)
============================================================

                  ┌─────────────────────────┐
                  │        clients          │
                  │─────────────────────────│
                  │ id           PK         │
                  │ fullname     NOT NULL   │
                  │ email                   │
                  │ phone                   │
                  │ city                    │
                  │ created_at              │
                  └────────────┬────────────┘
                               │ 1
                               │
                               │ N
                  ┌────────────┴────────────┐
                  │         orders          │
                  │─────────────────────────│
                  │ id           PK         │
                  │ client_id    FK→clients │   ON DELETE CASCADE
                  │ order_date   NOT NULL   │
                  │ status       ENUM       │
                  └────────────┬────────────┘
                               │ 1
                               │
                               │ N
                  ┌────────────┴─────────────┐
                  │       order_items        │
                  │──────────────────────────│
                  │ id           PK          │
                  │ order_id     FK→orders   │   ON DELETE CASCADE
                  │ product_id   FK→products │   (NO CASCADE — προστασία ιστορικού)
                  │ qty          NOT NULL    │
                  │ unit_price   NOT NULL    │   (snapshot τιμής στιγμής παραγγελίας)
                  └────────────┬─────────────┘
                               │ N
                               │
                               │ 1
                  ┌────────────┴────────────┐
                  │        products         │
                  │─────────────────────────│
                  │ id           PK         │
                  │ sku          UNIQUE     │
                  │ name         NOT NULL   │
                  │ price        DEC(10,2)  │
                  │ stock        INT        │
                  └─────────────────────────┘

Σχέσεις (συντομογραφία):
  clients (1) ──< orders (N) ──< order_items (N) >── (1) products

Λογική κανόνων:
  • Διαγραφή πελάτη → διαγραφή των παραγγελιών του (CASCADE).
  • Διαγραφή παραγγελίας → διαγραφή των γραμμών της (CASCADE).
  • Διαγραφή προϊόντος → ΑΠΟΡΡΙΠΤΕΤΑΙ αν υπάρχει σε γραμμές
    (διατηρούμε ιστορικά δεδομένα παραγγελιών).
  • unit_price στο order_items: snapshot — δεν αλλάζει αν αλλάξει η
    τιμή του προϊόντος αργότερα. Σωστή πρακτική σε e-commerce.
